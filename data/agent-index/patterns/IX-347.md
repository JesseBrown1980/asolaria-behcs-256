# The Orchestrator Loop

```
1. PLAN    → Write to memory (project_orchestrator_plan.md)
2. PID     → registerSpawnPid() for each agent
3. DISPATCH → Launch agents with EXACT IX numbers and type folders
4. WAIT    → Don't duplicate agent work
5. VERIFY  → Check files exist, catalogs updated, search works
6. DESPAWN → despawnPid() for each completed agent
7. MEMORY  → Update plan with results, mistakes, next phase
8. INDEX   → Create IX entries for mistakes and patterns discovered
9. GROW    → The system now knows more. Next spawn inherits everything.
```

This loop is the orchestrator's discipline. Skip any step and the system degrades:
- Skip plan → agents research randomly (IX-345)
- Skip PID → no tracking of who did what
- Skip verify → broken entries in index
- Skip despawn → stale PIDs, confusing history
- Skip memory → next session starts from zero
- Skip index → mistakes repeat (IX-168 collapse)

Cross-colony: both Gaia (IX) and Liris (LX) use the same loop. LX-249 (self-indexing rule) + IX-321 enforce it.
