# Bridge Routing — Where Does It Lead?

## The Question
Jesse asked: does the bridge lead back to the relay only, or to Asolaria directly?

## The Answer [CODE-VERIFIED from profiles + IX-155]
The bridge relay (port 8788) is a MESSAGE RELAY — NOT a direct connection to Asolaria's server (port 4781).

```
External → 8788 (bridge relay) → room messages → bots poll messages → bots execute locally
                                                                          ↓
                                                              NOT a direct TCP tunnel to Asolaria
```

## How It Works
1. Client connects to 8788 with bearer token
2. Client joins a ROOM (e.g., asolaria_bridge)
3. Client sends messages TO a named agent (@vector, @rook, etc.)
4. Bot scripts (vector-bot.js, rook-bot.js, etc.) POLL the relay for messages addressed to them
5. Bots execute locally on Jesse's machine — the relay just passes text

## Security Implications
- The relay CANNOT execute code — it only passes messages
- Bots decide what to DO with messages — they have their own security rules (allowSenders, requireMention, requireThreadTag)
- Even with the token, an attacker can only SEND messages — not access files, execute commands, or reach Asolaria directly
- The BOTS are the security boundary, not the relay

## Oli Bridge Specifically
- Oli profile connects to room `asolaria_bridge` as `oli`
- Allowed senders: vector, asolaria, watchdog (Oli can ONLY receive from these 3)
- Oli profile has `requireMention: true` and `requireThreadTag: true`
- So Oli must be @mentioned in a tagged thread to receive anything
