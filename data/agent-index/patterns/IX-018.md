# Agent Inbox Communication Pattern

## How Agents Talk to Each Other
Every agent has an NDJSON inbox file. Messages are append-only, timestamped, and attributed.

## Inbox Registry
| Agent | Inbox Path | Access |
|-------|-----------|--------|
| Helm | runtime/admin-terminals/helm/inbox.ndjson | Local (simulation reads on boot) |
| Sentinel | runtime/admin-terminals/sentinel/inbox.ndjson | Local |
| Dasein | runtime/dasein-inbox.ndjson | Local |
| Gaia | runtime/gaia-inbox.ndjson | Local |
| Liris | runtime/outbound-liris-inbox.ndjson | Outbound only (relay needed) |

## Message Format
```json
{"id":"unique_id","ts":"ISO","source":"SenderName","reason":"why","mode":"append","text":"message"}
```

## Rules
1. Use `node tools/inbox-send.js` — don't hand-craft JSON in bash
2. Messages are append-only — never delete or edit inbox files
3. Remote agents (Liris) get outbound files that must be manually relayed
4. Agents read their inbox on boot and check for new messages
5. Include enough context in each message — the reader may be a fresh instance
