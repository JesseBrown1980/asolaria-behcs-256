# Running Constructs — Services with PIDs

A new category in the index: **constructs** that are ALIVE — running processes with PIDs that agents depend on.

## What Makes a Construct
| Field | Purpose |
|-------|---------|
| pid | Process ID — is it alive? |
| type | What kind: broker, relay, server, agent |
| necessity | needed (never kill) or disposable (can replace) |
| agents | Who depends on this construct |
| chain | What IX entries it connects to |

## How Gaia Should Use This
When booting, check: are the needed constructs alive?
```bash
# Quick PID check
tasklist | grep -E "20992|18144|17988|12776|25576"
```
If any needed construct is dead, restart it BEFORE doing other work.
