# Device Registry

Every device in the network gets an index entry. The index maps: device → owner → agents → projects → sub-colony.

## Jesse's Sovereign Devices (FULL INDEX access)

| Device | Owner | Agents | Index Level |
|--------|-------|--------|-------------|
| Desktop (C:\Users\acer) | Jesse | Gaia, Dasein, Helm, Sentinel, Vector, Rook, Forge | FULL — master index |
| Samsung Galaxy S24 FE | Jesse (Falcon) | Falcon, phone agents | FULL — bidirectional mirror |

These two devices hold the COMPLETE civilization index. Jesse can access everything from either at any time.

## Sub-Colony Devices (SCOPED INDEX access)

| Device | Owner | Agents | Index Level |
|--------|-------|--------|-------------|
| Rayssa's PC (C:\Users\rayss) | Rayssa | Kuromi/Liris | SCOPED — own sub-colony index only |
| Felipe's phone | Felipe | TBD | SCOPED — own sub-colony index only |

Sub-colonies get their OWN indexing system (same IX format) but NOT the master civilization index. They can reference shared entries that Jesse explicitly exports.

## Why
- Jesse created everything. His devices = sovereign access.
- Sub-colonies are autonomous but scoped — they build their own knowledge.
- Future Asolaria users download and get a blank index template, build their own.
- The master index on Jesse's devices knows WHERE all sub-colony indexes are, but sub-colonies don't see the master.
