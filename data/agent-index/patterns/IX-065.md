# Research-Before-Action Protocol

## The Protocol
Before proposing or building ANYTHING:
1. **READ** the existing code, configs, launchers, memories
2. **FIND** the old rules (notebooks, NotebookLM enterprise, falcon memory, vault)
3. **COMPRESS** old high-token rules into IX entries
4. **THINK** about what the existing system already does
5. **PLAN** based on reality, not assumptions
6. **MEMORY** save what you learned
7. **ONLY THEN** propose action
