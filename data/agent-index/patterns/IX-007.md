# Index Types as Civilization Constructs

Each index type becomes a BUILDING or DISTRICT in the Asolaria civilization simulation.

## Proposed Constructs

| Type | Construct | Purpose | Who Maintains |
|------|-----------|---------|---------------|
| skill | **Skill Tower** | Known abilities, verified techniques | Any agent who learns one |
| tool | **Tool Forge** | External tools, APIs, commands | Forge, Dasein |
| pattern | **Pattern Library** | Recurring workflows, orchestration loops | Gaia, Helm |
| mistake | **Mistake Archive** | What went wrong + how to avoid | Any agent who makes one |
| plan | **War Room** | Active plans, labor divisions, timelines | Gaia (orchestrator) |

## How Agents Use Constructs
1. Agent boots → reads CATALOG.md
2. Gets task → searches catalog by tags
3. Finds relevant IX chain → follows it (loads 3-5 small entries)
4. Discovers reverse links → surfaces related knowledge
5. Completes task → writes new IX entries back to the right construct
6. SwarmDesk 3D view shows which constructs are "hot" (recently accessed)

## Helm's Role
Helm creates these as civilization entities in the simulation. They appear in Madness Interactive / SwarmDesk. Real buildings with real function — not just metaphors.
