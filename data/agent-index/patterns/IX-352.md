# IX-352 — Era-Based Index Division

## The Problem

At 351+ IX entries and growing, catalogs get heavy. Sub-catalogs helped (Phase 2.5) but they'll bloat too. An agent loading CATALOG-ARCHITECTURE.md (23 entries) is fine. At 50+ entries per sub-catalog, we're back to the IX-168 collapse pattern.

## The Rule: Max 20 Entries Per Sub-Catalog

When any sub-catalog exceeds 20 rows, split it by ERA:
- **Era 1** (IX-001 to IX-100): Genesis, founding, first architecture
- **Era 2** (IX-101 to IX-200): Research swarm, security excavation, awakening
- **Era 3** (IX-201 to IX-300): Federation, transport, colony convergence
- **Era 4** (IX-301+): Restructure, deep audit, operational indexing, dual orchestration

Each era gets its own sub-sub-catalog:
```
patterns/
  CATALOG-ARCHITECTURE.md          → router (4 lines pointing to eras)
  CATALOG-ARCHITECTURE-ERA1.md     → entries 001-100
  CATALOG-ARCHITECTURE-ERA2.md     → entries 101-200
  CATALOG-ARCHITECTURE-ERA3.md     → entries 201-300
  CATALOG-ARCHITECTURE-ERA4.md     → entries 301+
```

## Why Eras Work

1. Agent loading "architecture patterns from the founding era" reads ~10 entries, not 50
2. New entries always go into the CURRENT era — no re-sorting old data
3. Eras are IMMUTABLE once closed — Era 1 never changes, reducing conflict
4. Search still works — agentIndexStore scans IX-*.md files, not catalogs
5. Archaeology stays accessible but doesn't bloat operational catalogs

## When to Split

- Sub-catalog > 20 rows → split by era
- Era > 25 entries in one sub-catalog → something is miscategorized, review tags
- New era starts when IX numbers cross a century boundary (100, 200, 300, 400)

## Reduction Chain

This is the same principle at every level:
- CATALOG.md (682 lines) → thin router (40 lines) — DONE
- Type catalogs (100+ rows) → sub-catalogs (~20 rows each) — DONE
- Sub-catalogs (20+ rows) → era splits (~10 rows each) — NEXT when needed
- spawnContextBuilder loads MAX 12-20 entries per agent — ENFORCED

The weight stays constant. The knowledge grows. Compression wins.
