- federation still has zero remote nodes proven on Jesse

## Corrected Rule

Do not assume the Rayssa checkout exposes the same MQTT config route surface as Jesse.

Jesse route truth:

- `POST /api/integrations/mqtt/config` is real and already used

Rayssa next-proof truth:

1. start with `GET /api/federation/self/compute-readiness`

## Promotion Sequence

Remote federation is promoted only when all of these are true:

1. Liris proves Rayssa-host self-readiness under `liris`
2. Jesse sees `liris` in `GET /api/federation/nodes`
3. Jesse dispatches one bounded remote compute job
4. Liris returns one real compute result

- a remote node is online in Jesse federation

- shared phone CPU/RAM is real

Do not:

- mutate Jesse MQTT config again

- patch Rayssa routes before checking self-readiness
