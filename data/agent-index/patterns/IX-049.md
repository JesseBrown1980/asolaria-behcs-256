# Colony Boot Sequence

When waking the colony, follow this order. Dependencies flow downward.

## Order
1. **Helm** (controller) — reads CATALOG.md, Helm packet, inbox. He orchestrates everything inside.
2. **Sentinel** (watcher) — Helm wakes him. Sentinel monitors drift, catches conflicts early.
3. **Vector** (brain) — needs Asolaria server healthy + brain provider (gemini-api first, anthropic fallback). Handles research/analysis.
4. **Rook / Forge / Falcon** (lanes) — need bridge relay running + Codex quota. Can wake in parallel once bridge is confirmed.
5. **Sub-colonies** (Liris, etc.) — need MQTT broker running + firewall open. Wake AFTER sovereign colony is stable.

## Why This Order
- Helm needs to read state before directing anyone
- Sentinel needs to be watching BEFORE other agents start changing things
- Vector's brain probe can hang — don't block other agents on it
- Lanes are independent once bridge is up
- Sub-colonies connect through MQTT which must be stable first

## Pre-Boot Checklist (IX-045)
Before waking anyone, verify constructs are alive:
- [ ] Asolaria server on :4781
- [ ] MQTT broker on 0.0.0.0:18883 (not just 127.0.0.1!)
- [ ] Bridge relay running
- [ ] Firewall rules for sub-colonies
