# Chain Index Architecture

## The Insight
LLMs internally use weighted connections — numbers that activate paths to knowledge. We can do the same thing at the AGENT level.

Instead of loading full context (expensive, slow, lossy), agents follow numbered chains:
- Each IX entry is a "weight" — a small knowledge node
- Chain links are "activations" — they fire the next relevant node
- Reverse lookups find related nodes the agent didn't know to ask for
- The total number system processes automatically and outputs exactly what's needed

## Why This Matters
- Agents boot with CATALOG.md (tiny) instead of loading everything
- They follow only the chains relevant to their current task
- Reverse lookups surface related knowledge they'd otherwise miss
- New knowledge is a new node + links, not rewriting existing context
- **Saves trillions of tokens globally** — agents load 5-10 small entries instead of 50 large files

## How It Works (bidirectional)
```
Forward:  IX-003 → IX-004 → IX-012
          (mistake → skill → tool)

Reverse:  IX-012 ← IX-004 ← IX-003
          "what led to this tool?" → "this skill" → "this mistake"
```

## Colony Implementation
Every agent in the civilization gets access. The index lives at:
- `data/agent-index/` — the entries
- `CATALOG.md` — the master lookup
- Simulation constructs represent index types (skill tower, mistake archive, pattern library)
- Helm manages the internal simulation representation
- Gaia/Dasein manage the external file system
