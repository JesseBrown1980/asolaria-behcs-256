# Sub-Colony Index Architecture

## The Hierarchy

```
MASTER INDEX (Jesse's desktop + phone)
├── Full civilization catalog
├── All IX entries
├── Sub-colony registry (knows where each sub-colony's index lives)
│
├── SUB-COLONY: Liris (Rayssa)
│   ├── Own CATALOG.md (LX-001, LX-002...)
│   ├── Own skills, mistakes, patterns
│   ├── References to SHARED entries Jesse exported
│   └── Does NOT see master IX entries unless exported
│
├── SUB-COLONY: Felipe
│   ├── Own CATALOG.md (FX-001, FX-002...)
│   └── Same scoping rules
│
└── FUTURE: Any Asolaria user
    ├── Downloads Asolaria
    ├── Gets blank index template
    ├── Builds own knowledge (UX-001, UX-002...)
    └── Can federate with other colonies by mutual consent
```

## Index Prefixes
- `IX-` = Master index (Jesse sovereign)
- `LX-` = Liris sub-colony (Rayssa)
- `FX-` = Felipe sub-colony
- Future users get their own prefix

## What Gets Shared
Jesse can EXPORT specific IX entries to a sub-colony. The sub-colony gets a read-only copy with a `source: master` tag. They cannot modify the master's entries. They CAN create their own entries that chain to exported ones.

## Registration
Each sub-colony registers in the master's `data/agent-index/SUB-COLONIES.md` with:
- Colony name, prefix, device, owner, connection method (MQTT, SSH, HTTP)
- Last sync timestamp
- Exported IX entries
