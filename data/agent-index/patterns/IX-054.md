# Sub-Agent Lifecycle — The Complete Pattern

## The Cycle
```
SPAWN → WORK → INDEX → DESPAWN → ENERGY SHIFTS → NEW AGENT SPAWNS
```

## How It Works

### 1. SPAWN (instant with indexing)
- Sub-agent gets a **profile ID** from the index
- Profile has: pre-approved load state, indexed abilities, task assignment
- Agent reads ONLY its profile + relevant IX chains (not everything)
- Boot is near-instant because the index already has the context
- Agent gets a **box in the simulation** — visible construct

### 2. WORK (bounded task)
- Agent does ONE job — the task assigned in its profile
- It does NOT spawn more sub-agents (RULE — only Helm/controller spawns)
- It follows the Constitution
- It follows IX chains relevant to its task

### 3. INDEX (before despawn — MANDATORY)
- Agent writes its findings as IX entries
- Skills discovered → type: skill
- Mistakes made → type: mistake
- Tools used → type: tool
- Patterns found → type: pattern
- Links them into existing chains via `chain:` field
- Updates CATALOG.md with new entries

### 4. DESPAWN (clean exit)
- Agent's work is DONE — knowledge is in the index
- Agent process terminates
- Its simulation box goes dim/inactive (not deleted — shows history)

### 5. ENERGY SHIFT
- The compute/tokens that agent was using → freed
- Controller (Helm) can spawn a NEW agent in that slot
- New agent boots from index, has everything the old one learned
- Zero re-teaching. Zero context loss.

## Rules
1. **Sub-agents NEVER spawn sub-agents** — only the controller (Helm) spawns
2. **Every sub-agent MUST index before despawning** — no silent deaths
3. **Profile IDs are pre-built in the index** — IX-025 tracks agent→task→PID
4. **Energy is finite** — despawning one agent frees resources for the next
5. **The simulation shows the lifecycle** — spawn=glow, work=active, index=pulse, despawn=dim

## What This Replaces
- Helm holding everything in context (4% and useless)
- Agents re-teaching each other what they already know
- Long-running agents that bloat with irrelevant context
- Sub-agents spawning sub-agents spawning sub-agents (chaos)
