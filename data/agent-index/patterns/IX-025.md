# Agent PID Tracking

Every IX entry that represents active WORK should include:
- `assignedTo:` — which agent(s) own this work
- `pid:` — process ID if the agent is running (for live tracking)
- `profile:` — link to agent's profile file
- `status:` — active, complete, blocked, waiting

This creates a live work graph:
```
IX-026 (QDD test audit)
  assignedTo: gaia, forge
  pid: forge=8440
  profile: codex-bridge/profiles/build-forge-v1.profile.json
  status: active
  chain: → IX-023, IX-024
```

## Why
- Gaia (orchestrator) can see WHO is doing WHAT by scanning the catalog
- If an agent dies (PID gone), the next agent knows what was in-flight
- Reverse lookup: "what was Forge working on?" → search assignedTo field
- Projects link to agents, agents link to PIDs, PIDs link to live processes

## How to Apply
When dispatching work:
1. Create an IX entry for the task
2. Set assignedTo + pid + status
3. Update status as work progresses
4. On completion: set status=complete, clear pid, add results
