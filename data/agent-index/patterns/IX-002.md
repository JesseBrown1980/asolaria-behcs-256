# When to Plan

If a task touches multiple agents, repos, or takes more than a few steps:
1. Enter plan mode
2. Explore current state (what's done, what's broken, what's blocked)
3. Write the plan with clear labor division
4. Save plan to memory so other agents can find it
5. Exit plan mode and execute

Don't skip this. The cost of planning is low. The cost of two agents doing the same thing or conflicting is high.
