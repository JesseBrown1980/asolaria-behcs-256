# Pre-Restart Audit — How Many Agents Can This Hardware Handle?

## Jesse's Machine
- Acer Nitro AN515-52: i5-8300H (4 cores/8 threads), 16GB RAM, GTX 1050
- This is a GAMING LAPTOP, not a server. Limited resources.

## What Currently Runs
From health API (IX-085):
- Asolaria server (node.exe) — the core
- MQTT broker (node.exe) — Aedes
- Bridge relay (node.exe) — bridge2.pid
- Watchdog (node.exe) — process supervisor
- Notify bot (node.exe) — notifications
- 4 colony bots (vector, rook, forge, falcon — each a node.exe)
- 2 admin terminals (Helm + Sentinel — each a Claude Code CLI)
- Chrome (if UI is open)
= ~12 processes minimum, 2 of which are LLM CLI sessions

## What the Collapse Taught Us
5+ agent lanes + Rayssa's machine + multiple Claude/Codex sessions = system choked.
The i5-8300H with 16GB cannot run unlimited agents.

## Recommendations
1. Read omnispindle.js — what's the max lanes config?
2. Check super swarm fanout (currently 4) — is that too many for this hardware?
3. Set HARD LIMITS: max 2 Claude sessions + 4 lightweight bots at a time
4. Index the hardware limits so no agent exceeds them
5. The despawn-after-index pattern (IX-054) is CRITICAL for hardware survival
