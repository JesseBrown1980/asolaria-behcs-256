# IX-317 — Voice Injection Architecture

## Jesse's Vision

Normal system:
```
Microphone → Audio → Transcription Service → Human reads text
```

Asolaria-enhanced (stealth listen + speak):
```
Microphone → Audio ──→ [AGENT PRE-PROCESS] ──→ Transcription
                              ↓                      ↓
                    Agent hears raw audio     Agent reads transcript
                              ↓                      ↓
                    Agent generates response ←────────┘
                              ↓
                    [TTS INJECTION before human hears]
                              ↓
                    Human hears agent voice + sees transcription text
```

## What's Missing (To Complete the Pipeline)

1. **Orchestrator skill** — a single skill that chains: capture audio → transcribe → route to agent → agent responds → TTS → inject back
2. **Agent audio pre-processor** — hook into the audio stream BEFORE transcription so the agent can hear tone/context
3. **Multi-channel meeting router** — route different speakers to different agents
4. **Stealth mode flag** — toggle whether the human hears the agent's TTS injection or just sees text
5. **PID tracking for voice sessions** — which agent is listening to which audio channel

## Implementation Order

1. Wire voiceConnector + geminiLiveConnector into a single `voicePipeline.js`
2. Create `skills/voice-meeting-inject/skill.json` with the 6-step chain
3. Add voice session PID to spawnContextBuilder (Falcon role gets voice tasks)
4. Create MQTT topic `asolaria/voice/{sessionId}` for cross-machine audio routing
5. Wire caption bridge → agent inbox → agent response → TTS output

## The Abacus Insight

Jesse saw Abacus AI voice recorder doing: input → processor → divide video/audio → process audio → transcription → human reads.
Asolaria's enhancement: INJECT at every divide point. The agent doesn't just READ the transcription — it HEARS the audio, SEES the video, READS the text, and can SPEAK BACK before the human even processes it.
