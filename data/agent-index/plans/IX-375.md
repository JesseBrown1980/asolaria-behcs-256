## Ideas to Make It Faster

### 1. BOOT-LOAD OPTIMIZATION
Problem: spawnContextBuilder reads IX files from disk on every spawn.
Fix: Cache the typed folder contents in memory with TTL (agentIndexStore already does this).
Impact: Spawn goes from ~200ms disk reads to ~5ms cache hit.

### 2. MQTT INDEX SYNC (no more ZIP)
Problem: Cross-colony sync still requires manual ZIP exchange.
Fix: When an IX/LX entry is created, auto-publish summary to MQTT sync topic. Other colony stores cross-ref automatically.
Impact: Real-time index convergence. Zero human relay.

### 3. EMBEDDING-FIRST SEARCH
Problem: agentIndexStore uses keyword scoring. Good but misses semantic matches.
Fix: For spawn context, query semanticKnowledgeStore FIRST (vector similarity), fall back to keyword.
Impact: Agents find relevant entries they'd miss with keywords alone.

### 4. ERA-GATED LOADING
Problem: Sub-catalogs still load ALL entries for a type.
Fix: When sub-catalog > 20 rows, split by era. Agent loads CURRENT era by default, loads older eras only if chaining.
Impact: Constant load time regardless of index growth.

### 5. COMPUTE-DISPATCH FOR HEAVY QUERIES
Problem: Semantic search costs Gemini API tokens.
Fix: Cache embedding results. Cross-machine dispatch: if Liris has cached embeddings, query her instead of re-embedding.
Impact: Token cost reduction + faster response for repeated queries.

### 6. PRE-COMPUTED SPAWN PACKETS
Problem: buildSpawnContext runs fresh every time.
Fix: Pre-compute spawn packets for each role on index change. Store as JSON. Load instantly on spawn.
Impact: Spawn goes from ~500ms to ~1ms file read.
