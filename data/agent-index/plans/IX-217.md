# Kuromi Governance Plan

## Evidence

- `reports/helm-controller-packet-latest.md` says `Kuromi` is Rayssa's computer icon while `Liris` is the AI identity.
- `runtime/local-claude-profile.md` repeats the same split.
- The Constitution says persona naming is not the same thing as canonical node identity, external AI is untrusted by default, and no participant may self-promote.
- `tools/gaia-liris-bridge.js` exposes an unauthenticated `0.0.0.0:4799` message bridge, so any old Kuromi-side autonomy path must be audited before being trusted.

## Plan

1. Keep `Liris` as the named AI/admin identity.
2. Define `Kuromi` as a lower-permission companion persona inside the Liris sub-colony.
3. Give `Kuromi` a profile and memory anchors for fast reload.
4. Keep `Kuromi` off bridge-token, sender-policy, startup, and secret paths until the side-channel audit is complete.

## Why It Matters

This preserves Rayssa's requested name without collapsing the identity boundary between the device/window layer and the AI/admin layer.

It also reduces the risk that an old side channel or launcher path silently hijacks authority.
