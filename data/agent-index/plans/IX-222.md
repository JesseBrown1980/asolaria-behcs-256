# Temporary External-Node Connection Order

## Order

- Keep the temporary/unlicensed connection between the external nodes in place while QDD is active.
- Do not dismantle that path before QDD is fully ready and Jesse has personally confirmed the visual status.

## Why

- Current infrastructure cannot cheaply or reliably rebuild those connections by launching large agent waves just to re-establish the path.

## Limits

- This is a preservation exception, not an authority expansion.
- Do not use this order to add startup automation, expose secrets, or promote external nodes into higher-permission roles.

## Inspection Use

- Preserve the current external-node path while auditing which bridge, token, sender-policy, relay, or side-channel surface is actually carrying QDD coordination.
