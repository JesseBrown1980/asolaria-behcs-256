# The Great Cleanup — Plan

## Philosophy
Like cleaning an archaeological site: document what's there FIRST, then carefully remove debris layer by layer. Never bulldoze — you lose evidence. But don't leave collapsed buildings standing either.

## Phase 1: EXCAVATE (research only, no changes)
Map EVERYTHING that exists. For each item: what is it, who made it, when, is it still needed?

### 1A. Firewall Rules Audit
- List ALL inbound rules related to Asolaria/bridges/MQTT
- For each: who created it, when, for whom, still needed?
- Tag: KEEP / DELETE / SCOPE-DOWN

### 1B. Bridge Profiles Audit
- Read ALL profiles in codex-bridge/profiles/
- For each: who is this agent, still active, still needed?
- Special attention: Oli profiles, jarvis room, node templates
- Tag: KEEP / DELETE / ARCHIVE

### 1C. Bridge Token Audit
- Who has the bridge token (40ecab3963...)?
- Was it shared with Oli, Connor, Brian?
- Should it be rotated?

### 1D. Naming Confusion Audit
- Find all "this-codex" / "other-codex" / "that-codex" references
- Map to canonical names: vector, rook, forge, falcon
- Find all aliases and document which is canonical

### 1E. Tunnel Audit
- Cloudflare tunnels: any persistent ones still configured?
- Tailscale: what's connected?
- SSH tunnels: any port forwards still active?

### 1F. Port Usage Audit
- What ports are actually listening? (netstat)
- What ports SHOULD be listening?
- What ports have firewall rules but nothing listening?

## Phase 2: DOCUMENT (index everything found)
- One IX entry per finding
- Each tagged: KEEP / DELETE / SCOPE-DOWN / ROTATE / ARCHIVE
- Full chain linking so the reasoning is traceable

## Phase 3: PROPOSE (the cleanup plan)
- Present to Jesse: "here's what I want to delete/change"
- Jesse approves each change individually
- No bulk deletes without review

## Phase 4: EXECUTE (with Jesse's approval)
For each approved change:
1. Index what we're about to do (IX entry: "deleting X because Y")
2. Make the change
3. Verify nothing broke
4. Index the result

## Phase 5: VERIFY
- Full security scan after cleanup
- netstat shows only expected ports
- Firewall has only approved rules
- Bridge has only active profiles
- One bridge token, rotated, known only to authorized parties
- MQTT broker correctly scoped

## What We Do NOT Delete
- Agent inboxes (IX-056 — append only)
- Notebook entries (Constitution says never delete)
- Git history (the archaeological record)
- IX/LX entries (the compressed language)
- Any config Jesse says to keep

## Estimated Scope
- ~10 firewall rules to audit
- ~16 bridge profiles to audit
- ~5 tunnel configs to check
- ~20 port/service mappings to verify
- Total: ~50 items to excavate, document, decide on

## The Goal
After cleanup: ONE bridge token (rotated), ONE set of firewall rules (scoped to known IPs), ONE canonical name per agent, ZERO ghost configs, ZERO myths. The black hole sealed properly with ONE controlled entry point.
