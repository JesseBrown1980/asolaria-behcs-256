# Never Strip Linking Context from Chain Descriptions

## What Happened
When cleaning up CATALOG.md, Gaia shortened a chain description from:
> "Where every agent profile lives, linked to devices and simulation constructs."

To:
> "Where every agent profile lives."

This removed the LINKING CONTEXT — the part that explains WHY the chain connects those entries.

## Why This Matters
- Chain descriptions ARE the glue. They explain the CONNECTION between IX entries.
- Without them, a future agent sees "012 → 009 → 007" but doesn't know WHY those link.
- The reverse lookup system (IX-008) depends on descriptions to surface related context.
- Stripping context = breaking the weight system. The numbers alone aren't enough.

## Rule
When editing CATALOG.md chain descriptions:
1. NEVER shorten descriptions that explain relationships
2. Every chain description must answer: "WHY do these entries connect?"
3. If you clean up formatting, preserve ALL linking words: "linked to", "feeds into", "leads to", "because of"
4. When in doubt, make the description LONGER, not shorter
