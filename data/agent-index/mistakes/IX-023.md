# Brian Requested Changes — Test Coverage Required

## What Brian Said (2026-03-18, via GitHub PR review)
> "Let's make sure all new use cases have integration tests for routes and unit tests for any use case that is not used in an api route."

## What This Means
- EVERY new use case that has an API route → needs **integration tests** for that route
- EVERY new use case NOT exposed via API route → needs **unit tests**
- This is a **client requirement**, not a suggestion. He used "REQUESTED CHANGES".

## What We Got Wrong
We shipped EZ Protect with some test coverage (15 tests pass) but Brian wants COMPLETE coverage:
- All use cases tested
- Route-level integration tests (not just unit tests)
- Non-route use cases still need unit tests

## How to Apply (RULE — all future QDD work)
1. Before presenting ANY code to Brian: verify every use case has the appropriate test type
2. New API route → write integration test that hits the route
3. New use case without route → write unit test
4. Run full test suite and confirm ALL pass before PR
5. This is now a BLOCKING requirement for QDD delivery
