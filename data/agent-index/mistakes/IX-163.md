# Bridge Token Rotation — Relay Must Restart

Token was rotated in bridge-token.txt but the relay process (bridge2.pid) loads the token into MEMORY at startup. Changing the file doesn't affect the running process. Same pattern as IX-048 (MQTT broker bind not inherited).

Fix: restart the bridge relay via bridgectl.ps1 after token rotation.

But: restarting the relay disconnects ALL 7 currently connected bots (vector, rook, forge, falcon, watchdog, asolaria + ?). They'll reconnect via watchdog, but there will be a brief interruption.
