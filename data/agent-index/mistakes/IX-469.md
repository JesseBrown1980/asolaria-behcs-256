# IX-469 — Pre-Existing Test Failures: Root Causes

## Failure 1: index-kernel.unit.test.js (2 tests)

**Symptom:** "running profile uses manifest pointers" fails `false == true`. "staging validation rejects non-canonical payloads" fails `true !== false`.

**Root cause:** Test payload at line 161-177 is missing `validated: true` field. The staging gates expect this field to be present in a valid payload.

**Fix:** Add `validated: true` to `validPayload` in the test.

**Category:** Test defect, not code defect.

## Failure 2: aso.unit.test.js (1 test)

**Symptom:** "topics.json must exist" fails at line 226.

**Root cause:** aso.js caches `ASO_DATA_DIR` and `TABLES_DIR` as module-level constants at require time (lines 10-11). When the test harness swaps `ASOLARIA_INSTANCE_ROOT` and resets module cache, there's a path staleness risk — the constants capture the instanceRoot value at load time and never update.

**Fix:** Make `ASO_DATA_DIR` and `TABLES_DIR` dynamic getters instead of constants, so they re-resolve instanceRoot on each call.

**Category:** Module caching pattern defect.

## Failure 3: kernel-internals.unit.test.js (3 tests)

**Symptom:** "compile.buildCompiledDocuments is not a function" at lines 214, 261, 303.

**Root cause:** compile.js line 206 exports `buildSnippet` which is imported from textIds.js but never defined in compile.js scope. The phantom export may cause the module to partially fail or the function to be undefined.

**Fix:** Remove `buildSnippet` from compile.js exports — it belongs to textIds.js, not compile.js.

**Category:** Phantom export defect.

## Pattern

All three failures share one pattern: **stale or incorrect module state at test time**. Module-level constants that capture paths at load time, missing fields that are set during runtime but not in test fixtures, and exports that reference undefined symbols.

## Skill

When writing Node.js modules that will be tested with fresh instance roots: use dynamic path resolution (getters or functions), not module-level constants. When writing test fixtures: mirror the exact shape the runtime produces, including metadata fields like `validated`.
