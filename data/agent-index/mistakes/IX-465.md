# What went wrong

Two different audit formats were written into one file:

- `src/gateway/auditLog.js` uses a hash-chained gateway audit contract
- `middleware/attribution.js` appends plain request-attribution rows

Both targeted `data/gateway-audit.ndjson`.

The gateway verify pass therefore encounters the first non-chain row at entry `1146`, reports `prevHash mismatch`, and locks control writes on `4791`.

# Why it matters

This failure looks like general corruption, but it is actually a deterministic format collision.
If that is not recognized, operators can waste time on repeated restarts while the authority gate keeps relocking.

# Correction

Keep the two responsibilities separate:

1. gateway authority events stay in the hash-chained gateway audit file
2. main-app request attribution goes to its own append-only attribution log
3. the mixed historical file must be preserved, split, and reverified before restart claims

# Rule

Never write raw request-attribution rows into a file that the gateway verifies as a hash chain.
