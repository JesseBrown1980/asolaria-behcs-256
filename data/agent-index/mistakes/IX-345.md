# What went wrong

Gaia launched 4 research agents without:
1. Writing the plan to memory FIRST
2. Registering PIDs BEFORE dispatch
3. Defining verification checklist BEFORE launch
4. Specifying what IX entries each agent should create

Result: agents researched broadly but didn't auto-index. Orchestrator had to manually create IX entries from agent reports.

# How to avoid

Before EVERY agent dispatch:
1. Write plan to project_orchestrator_plan.md (what agents, what PIDs, what IX entries)
2. Register PIDs via registerSpawnPid()
3. Define verification checklist (what files, what catalogs, what searches)
4. Tell agents EXACTLY which IX numbers to create and which type folders
5. THEN dispatch

The rule: plan → memory → PID → dispatch → verify → despawn. Never skip steps.
