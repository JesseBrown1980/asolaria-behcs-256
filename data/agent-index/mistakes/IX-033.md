# Cross-Machine Auth Setup Delay

## What Happened
Getting the QDD repo to Liris was blocked for ~30 minutes because:
1. Gaia didn't read `git remote -v` directly (could have done it immediately — IX-028)
2. MQTT broker was down, so couldn't relay directly
3. Tried to ask Helm (who was at 4% context and unresponsive)
4. Tried vault API (unreachable at that moment)
5. Finally just read the git config locally

## Lesson
- Always check local resources FIRST before asking other agents
- If MQTT is down, don't keep trying — switch to WhatsApp relay immediately
- Helm at low context is effectively offline — don't depend on him for urgent info
- Bitbucket (not GitHub!) — the QDD repo platform assumption was wrong

## How to Apply
When an agent needs repo/credential info:
1. `git remote -v` in the repo directory (instant)
2. Check vault via Asolaria API (if up)
3. Ask Jesse directly (last resort)
Never wait for Helm when he's at low context.
