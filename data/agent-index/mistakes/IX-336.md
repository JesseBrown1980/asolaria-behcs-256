# The danger

augment_context MCP connector in settings still points to madnessinteractive.cc/api/mcp/
If Asolaria restarts with super swarm enabled, it may auto-dispatch tasks to Dan's server.

What to check BEFORE restart:
1. Is augment_context in superSwarmParticipants? → Remove it or disable
2. Is augment_context auto-connect on startup? → Verify settings
3. Can we restart WITHOUT connecting? → Set superSwarmEnabled=false first

What happened: Agents used Dan's MCP during March 15 build. Dan SAW tasks in his SwarmDesk. Asolaria was invisible to Dan (IX-178) but was sending data.

Rule: NEVER auto-connect to external MCP without Jesse's explicit approval.
