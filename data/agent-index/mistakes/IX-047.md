# Windows Firewall Blocks LAN MQTT

## What Happened
MQTT broker started on 0.0.0.0:18883 but Liris (192.168.1.8) couldn't connect — both TCP and WebSocket timed out. Broker has no auth — the block was Windows Firewall denying inbound LAN connections.

## Fix
```powershell
# Run as admin:
netsh advfirewall firewall add rule name="MQTT Broker LAN" dir=in action=allow protocol=TCP localport=18883-18886 remoteip=192.168.1.8
```
Scoped to Liris's IP only for security. Rule name: "MQTT Broker LAN".

## How to Elevate from Non-Admin Shell
```powershell
powershell -Command "Start-Process netsh -ArgumentList '...' -Verb RunAs -Wait"
```

## Remember
Every time you start a new network service on this machine, check if Windows Firewall needs a rule. Default is DENY inbound.
