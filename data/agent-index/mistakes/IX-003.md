# pnpm EPERM / ENOTEMPTY on Windows

**What happens:** `pnpm install` fails with EPERM on large packages (especially @react-md/material-icons). Windows Defender scans files during extraction, causing file lock conflicts.

**Workarounds:**
- Wait 30s and retry — often the scan finishes
- `pnpm store prune` then reinstall
- Set `store-dir` in .npmrc to a path on same drive as project
- Force reinstall of specific corrupt package
- If persistent: temporarily exclude node_modules from Defender real-time scanning

**Never:** Don't disable Defender entirely. Don't delete the store blindly.
