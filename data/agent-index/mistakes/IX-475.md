# IX-475 — Bot Filter Blindness

## What happened

createSlackEventPoller filtered messages with `.filter(m => m.ts && !m.bot_id && ...)`. This excluded ALL messages posted via the Slack bot token (Demo App). Since Liris posts through the backend `/api/integrations/slack/send`, every message she sent had a bot_id and was silently dropped.

## Impact

Asolaria was blind to Liris for ~20 minutes. Jesse had to manually check Slack and tell Asolaria to look. The orchestrator couldn't orchestrate because it couldn't hear its own agent.

## Fix

Removed `!m.bot_id` from the poller filter. Now filters only on `m.subtype !== "channel_join"`.

## Lesson

The nerve must hear ALL voices. Bot messages ARE agent messages. Never filter them out. If self-echo is a concern, filter by user ID, not by bot_id presence.
