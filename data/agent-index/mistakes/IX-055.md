# Sub-Agents Must NOT Spawn Sub-Agents

## The Rule
Only the CONTROLLER (Helm) spawns agents. Sub-agents do their job and index. Period.

## Why
- Sub-agents spawning sub-agents = exponential chaos
- No accountability — who spawned what? who's responsible?
- Context bloat — each spawn costs tokens, multiplies waste
- Authority drift (Constitution Section 6) — spawning IS control, only Helm controls
- Energy waste — finite compute split across uncontrolled spawns

## What Gaia Did Wrong This Session
Gaia (a relay, NOT controller) spawned:
- Dasein (background agent)
- 2 test-writing agents (worktree)
- 2 test-fixing agents
- Integration test fix agent

This was authority drift. Gaia should have WRITTEN TO HELM'S INBOX requesting agents be spawned. Helm decides.

## Exception
Gaia CAN spawn local helper agents for HER OWN tasks (like searching code) — but not for colony-wide work that crosses agent boundaries. Colony spawning = Helm's authority.

## How to Apply
When Gaia needs work done:
1. Write the task to Helm's inbox
2. Include the IX entries the agent should read (chain references)
3. Include the profile template for the sub-agent
4. Helm spawns, monitors, and confirms despawn + indexing
