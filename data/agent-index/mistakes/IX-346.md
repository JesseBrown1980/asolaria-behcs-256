# What went wrong

After the deep audit found 231 unindexed items, Gaia spent time creating reference/archaeology entries (IX-322, IX-323, IX-324) instead of the operational skill/tool entries that agents actually need to USE the system.

Jesse corrected: "that is archaeology, not skills and tools and PIDs and abilities."

# How to avoid

Priority order for indexing:
1. **Skills** — HOW to use a capability (API endpoint, steps, config)
2. **Tools** — WHAT the tool does (ports, commands, parameters)
3. **Mistakes** — WHAT to avoid (with specific failure modes)
4. **Patterns** — HOW things connect (architecture, workflows)
5. **References** — WHAT exists (archaeology, history, context)

Always ask: "Can an agent USE this entry to DO something?" If not, it's archaeology — useful but lower priority than operational entries.
