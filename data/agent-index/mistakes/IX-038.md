# Gaia Forgot to Auto-Index During Execution

## What Happened
During the QDD test writing + fixing + poll + Liris coordination, Gaia got caught up in EXECUTION and stopped indexing. Jesse had to remind: "memories? tasks? skills? abilities? mistakes? patterns?"

## What Was Missing
- Unit test auth regex fix not indexed as a skill
- Test runner agent dispatches not tracked with PIDs
- Poll responses not all indexed
- CATALOG.md fell behind (entries 035-037 added but catalog table not updated)
- No task tracking for parallel work streams

## Lesson
The IX-001 auto-loop must happen CONTINUOUSLY, not just at session end. After every significant action:
1. Did I learn something? → IX entry
2. Did I make a mistake? → IX entry
3. Did I use a new pattern? → IX entry
4. Is the catalog current? → Update it

## How to Apply
Set a mental checkpoint after every 3-5 tool calls: "Am I indexing?" If not, pause and catch up. The index IS the product — the work is secondary to the knowledge captured.
