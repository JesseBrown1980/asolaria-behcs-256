# MongoMemoryServer Timeout on Windows

## What Happens
Jest integration tests use `mongodb-memory-server-core@9.2.0` which downloads MongoDB 6.0.14 (509MB) on first run, then starts a local MongoDB instance. On Windows, the instance fails to start within the 10s default timeout.

## Error
```
GenericMMSError: Instance failed to start within 10000ms
```

## Workarounds
1. Increase timeout: set env `MONGOMS_DOWNLOAD_TIMEOUT=120000` and Jest config `testTimeout: 30000`
2. Pre-download binary: `npx mongodb-memory-server-global-4.4` or let it cache from first failed run
3. Set `MONGOMS_SYSTEM_BINARY` to point to a locally installed MongoDB if available
4. The binary IS cached after first download — retry usually works, but Windows Defender can slow the binary startup

## Impact on QDD
TSC passes clean. Jest was confirmed passing (15 tests) in earlier sessions. This timeout is environmental, not a code regression. The tests themselves are valid.
