# What's wrong

The MQTT bridge config (tools/gaia-liris-bridge.js line 5) points to 192.168.1.3:18883.
The current WiFi network assigns:
- Jesse: 192.168.15.189
- Rayssa: 192.168.15.170

The old 192.168.1.x network is gone. Federation silently fails because the IP doesn't resolve.

# How to fix

Liris runs:
```
curl -X POST http://127.0.0.1:4781/api/integrations/mqtt/config \
  -d '{"brokerUrl":"mqtt://192.168.15.189:18883","baseTopic":"asolaria/nodes/liris"}'
```

# How to avoid in the future

ALWAYS check IPs before checking encryption (IX-310).
Network changes = IP changes = federation breaks silently.
The first diagnostic on any federation failure: `ipconfig` / `ip addr` on both machines.
