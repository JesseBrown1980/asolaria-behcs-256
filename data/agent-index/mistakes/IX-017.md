# NDJSON Escaped Newlines Bug

## What Happened
When writing multi-line messages to inbox NDJSON using bash `echo`, literal `\n` in the text field gets stored as escaped characters. JSON.parse handles this fine, but the `read` display truncates at 120 chars so context is lost.

## Lesson
- Always use JSON.stringify (like inbox-send.js does) instead of hand-crafting JSON in bash
- The inbox-send.js tool handles this correctly — prefer it over raw echo/append
- When reading, some messages from older bash-appended entries may show as "unparseable" if the escaping was inconsistent

## How to Apply
Use `node tools/inbox-send.js` instead of raw bash echo for all inbox writes going forward.
