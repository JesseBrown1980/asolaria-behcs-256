The first live rollout slice is now real.

What is implemented:

- `routes/workerRouter.js` passes request/auth attribution into `dispatchWorkerTask`
- `src/connectors/workerRouter.js` preserves request context
- `src/connectors/workerRouter.js` acquires and releases tumbler leases for true `executeNow` runs on:
  - `worker_lane.local_codex`
  - `worker_lane.claude_max`
- `src/modelLaneTumbler.js` supports `queueIfBusy=false` so the router can fail fast on a busy lane instead of queueing work it cannot yet resume

What is intentionally not leased:

- `abacus` packet creation
- `symphony` issue submission
- any `executeNow=false` prompt-generation path

This keeps runtime enforcement truthful: only scarce live execution lanes consume tumbler capacity.

Verified behavior:

- non-executing worker dispatch still succeeds with `lease = null`
- direct tumbler smoke now grants first lease, throws a clean busy error on second no-queue request, and releases cleanly

- restart the main Asolaria runtime
- validate authenticated `/api/catalog/summary` and `/api/catalog/selectors`
- then add a small truthful selector summary surface in mobile/admin UI
