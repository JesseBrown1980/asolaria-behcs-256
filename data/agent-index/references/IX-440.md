- `192.168.1.43` ping returned `host unreachable` from `192.168.1.10`

- earlier TCP reachability to `192.168.1.43:443` also failed

## Operational meaning

The current failure is not "the app cannot find the device."

The current failure is:

1. the device/app fixture is healthy on COM9
2. the current LAN segment does not contain the configured sync target host
3. the likely intended fixture SSID is saved locally but not currently available

## Practical rule

Do not treat more blind app-side sync attempts on `Jesse_5G` as meaningful proof.

The next valid NovaLUM live action is one of:

1. make the intended NovaLUM LAN visible again and join it
2. bring the expected sync-server host back onto the current LAN
3. explicitly re-home the sync target under a dedicated mutation plan
