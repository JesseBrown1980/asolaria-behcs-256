Live-proof dependency classification:

Queue / SFTP:
- mostly locally recoverable
- local prerequisites already exist or are plausible on Jesse's machine: repo, Mongo runtime, web route, queue-server lane
- remaining dependency is a configured FTP/SFTP location plus one controlled import file
- classification: `local + configuration-sensitive`

EZ Protect:
- partly locally recoverable but still externally blocked
- local code, routes, and test evidence exist
- missing live proof depends on a deployed/authenticated environment, representative EZ event data, and permission-gate validation
- reviewer evidence also says real data had not yet been received and the exact device-to-data/API path was still not well defined
- classification: `mixed local/external, currently externally blocked`

Operational meaning:
- Queue is the easier lane to re-prove locally if a controlled import target can be recreated
- EZ Protect likely cannot be fully re-proved from local branch state alone without recovering or recreating the external auth/data/integration surface
