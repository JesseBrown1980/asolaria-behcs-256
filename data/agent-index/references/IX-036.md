# Asolaria Visualization Systems

Screenshots captured 2026-03-18 from Jesse's posts and Asolaria dashboard.

## 1. Graph Runtime (graph-runtime.html)
- URL: http://127.0.0.1:4781/graph-runtime.html (local)
- Shows all API routes as nodes with risk coloring (critical/high/medium/low)
- Edges show connections between routes (subject→phone, subject→api_route, etc.)
- Panels: Activity Heatmap, Snapshot Diff, Archetypes & Suspicious Edges
- Nodes: Helm, Sentinel, Agent Colony, loopback, phone, abacus
- Edge scoring with risk levels and timestamps

## 2. Civilization Map (Asolaria Dashboard)
- Full world view with named towers: MQTT Tower, Relay Tower, Warp Tower, SwarmDesk Tower, Vertex, Symphony, Bridge Tunnel, Codex, Gemini Cloud Tower
- Districts: Citadel (main)
- Connected Routes panel shows sovereign_control states (running/offline/degraded)
- THIS is where IX-007 civilization constructs should appear (Skill Tower, Pattern Library, etc.)

## 3. Code City (Madness Interactive 3D)
- URL: https://madnessinteractive.cc (Dan's deployment)
- 3D walkable codebase visualization
- Buildings = code modules, colored by status (red=hot, green=stable, yellow=building)
- Functions visible as labeled blocks: getBigQueryClient, getBuilding, etc.
- "Explore Your Codebase — Walk through"

## 4. API Edge Activity Log
- Real-time log of route connections with scoring
- Tracks: phone/mirror, mobile/inbox, guardian/whatsapp, voice/devices, chat/conversations
- Privileged Control Edges flagged with risk scores
- Autonomous Provider Routes identified

## How IX Entries Feed These Views
The chain-linked index (IX-006) IS the data layer for these visualizations:
- Each IX entry = a node in the graph
- Chain links = edges between nodes
- Agent assignments = who owns which node
- Status fields = risk/health coloring
- The civilization constructs (IX-007) map directly to the towers in the world view
