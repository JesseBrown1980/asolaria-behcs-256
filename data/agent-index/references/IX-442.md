On 2026-03-29, the NovaLUM host-side service gap was closed on the laptop.

## What changed

The laptop had already been re-homed to the sync target address:

- `192.168.1.43/24`
- gateway `192.168.1.1`
- DNS `208.67.220.220`, `208.67.222.222`

After that, the QDD sync-server was launched locally from:

- `D:\projects\QDD\ebacmap-master\apps\sync-server`

using explicit local overrides for:

- `SERIAL_PORT=443`
- `APP_ENV=local`
- `API_URL=http://localhost:3000`
- `APP_URL=http://localhost:3000`
- `MONGODB_URI=mongodb://localhost:27017/ebacmap-local-test?readPreference=primary&directConnection=true&ssl=false`
- `MONGOOSE_DEBUG=false`
- `USE_ENCRYPTION=false`

## What was proven

The sync-server startup logs now show:

- `STARTING SERIAL SERVER 443`
- `listening on port 443`

Artifacts:

- `D:\projects\QDD\ebacmap-master\reports\novalum-sync-server-20260329-140722.out.log`
- `D:\projects\QDD\ebacmap-master\reports\novalum-sync-server-20260329-140722.err.log`

Runtime confirmation:

- local listener exists on `::443`
- owning process is the new Node sync-server process
- `Test-NetConnection 192.168.1.43 -Port 443` now returns `TcpTestSucceeded=True`

## Significance

This is the first state where:

1. the NovaLUM device path is healthy through `NL2 INTERFACE ADAPTER -> COM9`
2. the laptop owns the configured sync target address `192.168.1.43`
3. a real QDD sync-server is listening on the configured port `443`

That means the previous blocker has moved again:

- no longer device discovery
- no longer missing host address
- no longer missing host listener

The next step is the bounded NovaLUM no-data sync proof against the live local server.

## Practical rule

Do not describe this as reverse ADB.

The preserved architecture is:

- serial bridge / NL2 adapter on `COM9`
- desktop sync app as the control surface
- local TCP sync-server on `192.168.1.43:443`

That is functionally a host-mediated relay path, but it is not Android ADB reverse.
