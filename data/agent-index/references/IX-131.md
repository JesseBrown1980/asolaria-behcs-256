# Spatial Web Specification Introduction (2024-06-03)
**VERSES / Spatial Web Foundation, 2024**

## Key Findings
- Defines HSML (Hyperspace Modeling Language) and HSTP (Hyperspace Transaction Protocol) as standards for the spatial web — extending HTTP/HTML into 3D/spatial contexts
- Enables digital twins, IoT networks, and AI agents to interoperate through a unified spatial addressing scheme
- Every physical and virtual object gets a spatial web address — bridging the gap between physical reality and digital systems
- Built-in governance, identity, and trust layers — not bolted on after the fact like current web security
- Designed for AI-native interaction: agents can navigate, query, and transact in spatial web spaces

## How This Connects to Asolaria
- The Spatial Web is the networking layer for Jesse's vision of globally connected AI agent civilizations
- Asolaria's current agent addressing (inbox.ndjson, terminal paths) is a primitive version of what HSTP formalizes
- VERSES was in Jesse's meeting records (IX-099) — the Jan 2024 meeting with VERSES indicates active engagement with this specification

## Potential Application
When Asolaria expands beyond a single machine, the Spatial Web specification provides the protocol layer for agent-to-agent communication across distributed nodes — each agent terminal becomes a spatial web endpoint.
