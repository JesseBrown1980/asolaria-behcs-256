# routes/mobile.js — 47 routes (mobile/phone remote control)

Mounted as `/api/mobile/*`. Uses `createMobileRouter(ctx)`. Auth: nearly all routes use `requireMobileAuth`. Desktop control routes add `requireMobileDirectControlEnabled` + `requireDesktopSuperMasterAuthority`.

## Route Groups

**Session & Settings (3):** `/session`, `/settings`, `/bootstrap` — mobile session state, runtime settings snapshot, bootstrap payload for first connect.

**Inbox & Tasks (2):** `/inbox`, `/task-ledger/tasks/:id` PUT — mobile inbox with pending approvals, task ledger update.

**Token (1):** `/token/rotate` — rotate mobile approval token.

**Screen Capture (4):** `/screen/status`, `/screen/displays`, `/screen/capture`, `/screen/image` — desktop screen mirroring to phone. Capture returns display topology + images.

**Desktop Control (7):** `/control/status`, `/control/diagnostics`, `/control/arm`, `/control/disarm`, `/control/click`, `/control/scroll`, `/control/type`, `/control/key` — full remote desktop input. All gated behind triple auth (mobile + direct-control-enabled + super-master).

**Bridge & Tunnel (2):** `/bridge/keeper/status`, `/tunnel/monitor/status` — phone bridge keeper and tunnel health.

**Phone Developer Options (4):** `/phone/developer-options/catalog`, `/phone/developer-options/status`, `/phone/developer-options/set`, `/phone/developer-options/sweep` — ADB developer option management.

**Help & UI (3):** `/help/recover`, `/ui/open`, `/ui/cleanup` — recovery actions, open Asolaria UI targets, cleanup managed Chrome windows.

**Power Management (6):** `/api/power/status`, `/api/power/shutdown`, `/api/power/restart`, `/api/power/abort`, `/api/power/wol/status`, `/api/power/wol/config`, `/api/power/wol/send` — remote shutdown/restart/abort + Wake-on-LAN.

**Windows (3):** `/windows` GET, `/windows/focus`, `/windows/action` — list/focus/action on desktop windows.

**Push Notifications (5):** `/push/public-key`, `/push/status`, `/push/subscribe`, `/push/unsubscribe`, `/push/test` — web push subscription management.

**Approvals (4):** `/approvals` GET, `/approvals/:id`, `/approvals/:id/approve`, `/approvals/:id/deny` — guardian approval queue from phone.

**Connections (1):** `/connections` POST — connection routing config.

## Notable Patterns
- Triple-auth stack on all destructive desktop control (click/scroll/type/key, power, window actions)
- Power management includes Wake-on-LAN for remote boot
- Push notification system for real-time mobile alerts
- Screen capture pipeline: capture → image URL → phone display
