## Practical rule

Do not spend more NovaLUM effort on `appsettings.json` as a live control surface until a different bootstrap path is proven.

The host-side priority order is now:

1. keep the laptop at `192.168.1.43` if this machine is meant to be the sync server
2. bring up the sync-server listener on port `443`
3. only then retest the bounded no-data NovaLUM connection proof
