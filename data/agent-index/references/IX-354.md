# 4 GNN Model Architectures Compared

All models live in `services/gnn-sidecar/models/`. All share the same GCN encoder backbone (2 GCNConv layers with ReLU + dropout) and the same 6-dim node features / 3-dim edge features defined in IX-344. All output per-edge anomaly probability [0,1]. All include a dummy GCNConv fallback so the file parses without torch_geometric.

## 1. EdgeLevelGNN (Baseline)

- **File:** `gnn_baseline.py`
- **Class:** `EdgeLevelGNN`
- **Init params:** `node_input_dim`, `hidden_dim`, `output_dim=1`
- **Architecture:** GCN encoder (conv1 → ReLU → 50% dropout → conv2) → concatenate [node_u, node_v] → MLP edge classifier (Linear → ReLU → Linear → Sigmoid)
- **Output:** Single sigmoid score per edge. > 0.5 = suspicious.
- **What makes it unique:** Simplest model. Direct binary sigmoid output, no class probabilities. BCE loss.
- **When to use:** First model to try. Baseline for comparison. Fast training.
- **Reported accuracy:** ~91.87% (implied from contrastive's +2.84% claim)

## 2. ContrastiveGNN

- **File:** `contrastive_gnn.py`
- **Class:** `ContrastiveGNN`
- **Init params:** `node_input_dim`, `hidden_dim`, `projection_dim=64`, `num_classes=2`
- **Architecture:** GCN encoder → edge MLP (with 30% dropout) → TWO heads:
  - **Projection head:** Linear → ReLU → Linear(projection_dim) → L2-normalize. Used for contrastive loss.
  - **Classifier head:** Linear → ReLU → 30% dropout → Linear(num_classes) → softmax.
- **Output:** Anomaly class probability via softmax. `return_projections=True` also returns normalized embeddings.
- **Loss:** Dual loss — `SupervisedContrastiveLoss` (SupCon, temperature=0.07) on projections + CrossEntropy on classifier. Both losses combined during training.
- **What makes it unique:** Learns embedding geometry where same-class edges cluster and different-class edges separate. The projection head is only used during training — at inference only the classifier fires.
- **When to use:** When class imbalance exists. Contrastive learning helps the model learn better representations even with few suspicious samples.
- **Reported accuracy:** 94.71% (+2.84% over baseline)

## 3. GSLGNN (Graph Structure Learning)

- **File:** `gsl_gnn.py`
- **Classes:** `GraphStructureLearner` + `GSLGNN`
- **Init params:** `node_input_dim`, `hidden_dim`, `num_classes=2`, `gsl_hidden_dim=32`
- **Architecture:** THREE stages:
  1. **GraphStructureLearner:** key/query transforms on node features → attention matrix → sigmoid → sparsify with learnable threshold → 50/50 blend with original adjacency.
  2. **Dual-branch GCN:** learned-graph branch (conv1+conv2) + original-graph branch (conv1_orig+conv2_orig) run in parallel.
  3. **Edge classifier:** concatenate edge embeddings from BOTH branches (4 * hidden_dim input) → 3-layer MLP → softmax.
- **Output:** Anomaly class probability via softmax. `return_weights=True` also returns learned adjacency matrix.
- **Extra methods:** `get_learned_graph()` for visualization, `get_edge_importance()` extracts per-edge importance from learned adjacency.
- **What makes it unique:** LEARNS A NEW GRAPH STRUCTURE from node features via attention, then combines it with the original. The dual-branch design means the model sees both the real topology and its own inferred topology. This is the most parameter-heavy model.
- **When to use:** When the original graph topology is noisy or incomplete. Best overall accuracy. Use when you need explainability (the learned adjacency shows what the model thinks connections should be).
- **Reported accuracy:** 96.66% (+4.79% over baseline) — BEST. ROC-AUC: 99.70%. FPR: 1.5%.

## 4. PrototypeGNN

- **File:** `prototype_gnn.py`
- **Class:** `PrototypeGNN`
- **Init params:** `node_input_dim`, `hidden_dim`, `num_classes=2`, `num_prototypes_per_class=3`, `temperature=0.1`
- **Architecture:** GCN encoder → edge projection MLP (2 * hidden_dim → hidden_dim → hidden_dim) → distance to learnable prototypes → softmax.
  - **Prototypes:** `nn.Parameter` tensor of shape [num_classes, num_prototypes_per_class, hidden_dim]. Xavier initialized.
  - **Distance:** Squared Euclidean distance, min over prototypes per class, divided by temperature.
- **Output:** Anomaly class probability via softmax over negative distances.
- **Extra:** `get_prototype_loss()` — optional separation loss (just cross-entropy on prototype distances).
- **What makes it unique:** Classification by PROXIMITY to learned prototype vectors, not by a classifier MLP. Each class has multiple prototypes (default 3) to capture diverse attack patterns. The model learns "what does a typical benign edge look like?" and "what does a typical suspicious edge look like?" as actual vectors in embedding space.
- **When to use:** When you need interpretability — you can inspect what the prototypes represent. Good for few-shot scenarios where new attack types emerge (just add prototypes).
- **Reported accuracy:** 94.24% (+2.37% over baseline)

## Quick Selection Guide

| Need | Model |
|------|-------|
| Fast baseline, first experiment | EdgeLevelGNN |
| Class imbalance, few suspicious edges | ContrastiveGNN |
| Best raw accuracy, noisy graph | GSLGNN |
| Interpretability, new attack types | PrototypeGNN |
| Production default | GSLGNN (if compute allows) or EdgeLevelGNN (minimal) |

## Shared Constraints

- All models require PyTorch + PyTorch Geometric (`pip install torch torch_geometric`).
- Dummy GCNConv fallback exists in all files — it is just `nn.Linear`, NOT a real GCN. Never use for real training.
- All trained via `train_baseline.py` (currently only wired for EdgeLevelGNN — other models need their own training scripts or modification of the trainer).
- Temporal split: 70/15/15, never shuffle.
