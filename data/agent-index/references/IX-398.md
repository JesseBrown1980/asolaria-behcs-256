# QDD Forced Runtime Baseline — pnpm 10 only

## Rule

QDD runtime baseline is `pnpm 10`.

Point final.

## Meaning

- `ebacmap-master` is the source of truth
- `packageManager`: `pnpm@10.32.0`
- `engines.pnpm`: `^10.0.0`
- pipeline baseline: `PNPM_VERSION=10.32.0`

## Enforcement

- any split branch, `pre-dev` worktree, helper lane, pseudo-live lane, rerouter lane, `dev`, `staging`, or `prod` packet must be treated as invalid if it drifts back to `pnpm 9`
- do not treat `pnpm 9` split-branch test results as authoritative for QDD promotion
- first repair is forward alignment to the `pnpm 10` source baseline

## Shorthand

- `QDD = pnpm10`
- `split drift to pnpm9 = invalid baseline`
- `align forward, do not reinterpret from stale runtime`

## Why

- reviewer split request was about `SFTP / queue` vs `EZ Protect`
- `pnpm 10` is runtime truth, not optional nostalgia
- keeping this rule short prevents later agents from re-arguing settled baseline facts
