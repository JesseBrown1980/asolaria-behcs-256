# routes/remaining.js — 67 routes (the catch-all core router)

Mounted as `/api/*`. Uses `createRemainingRouter(ctx)`. Auth: mixed (loopback-only, requirePhoneBridgeAccess, requireDesktopSuperMasterAuthority, open).

## Route Groups

**Startup & Brain Leader (3):** `/startup/brain-leader/run`, `/automation/brain-leader/select`, `/asolaria/runtime-actions` — brain probe, auto-select, runtime action list/run. Super-admin terminals GET is loopback-only.

**Settings (1):** `POST /settings` — massive handler (~500 lines) accepting all runtime setting mutations.

**Swarm (2):** `/swarm/status`, `/swarm/mode` — super-swarm coordination. Mode change is loopback-only.

**Company/Moltbook Signin (7):** `/company/primary-signin/*`, `/moltbook/account/*` — bootstrap, update, migrate-legacy, mark-registered, reveal. Identity management.

**Providers & Models (7):** `/providers/external/status`, `/providers/model-aliases`, `/providers/model-settings/*`, `/providers/brain/run`, `/providers/openai/run`, `/providers/codex/run`, `/providers/external/run` — model routing, cooldown, observed model prune/clear.

**Avatar (2):** `/avatar/state` GET/POST — avatar render mode and NPC state.

**Guardian (4):** `/guardian/approvals/decide`, `/guardian/approvals/webhook`, `/guardian/approvals/whatsapp/inbound`, `/guardian/alert/test` — approval pipeline + WhatsApp inbound hook.

**Memory (5):** `/memory` GET/DELETE, `/memory/search`, `/memory/compaction/proposal`, `/memory/compaction/apply` — conversation memory with compaction and clear (approval-gated).

**Chat (3):** `/chat` POST (main chat endpoint ~400 lines), `/chat/conversations`, `/chat/conversations/:chatId`.

**Notebook (7):** `/notebook/notes` CRUD, `/notebook/search`, `/notebook/maintenance/normalize` (loopback), `/notebook/google/mirror/*` — Google mirror sync.

**Skills (7):** `/skills` GET, `/skills/codex`, `/skills/index`, `/skills/index/rebuild`, `/skills/:id`, `/skills/reload`, `/skills/run` — full skill registry API.

**Local Ops & Browser (5):** `/local/ops/status`, `/local/ops/run`, `/browser/task`, `/automation/form-channel`, `/automation/chat-channel`.

**Chrome (4):** `/chrome/open`, `/chrome/profiles`, `/chrome/inspect`, `/chrome/screenshot`.

**Pad & Gateway (3):** `/pad/open`, `/pad/run`, `/gateway/remediation/start-sandbox`.

**OpenClaw (1):** `/openclaw/action` — game/simulation action dispatch.

## Notable Patterns
- Largest route file in the system (~3300 lines)
- `POST /settings` alone is ~500 lines handling every runtime toggle
- `POST /chat` is ~400 lines — the primary conversation endpoint
- Loopback-only gates on: swarm/mode, brain-leader/select, super-admin terminals, notebook maintenance
- Guardian approval flow spans 4 endpoints including WhatsApp webhook
