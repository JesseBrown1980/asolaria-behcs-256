# routes/phone.js — 31 routes (phone hardware bridge)

Mounted as `/api/phone/*`. Uses `createPhoneRouter(ctx)`. Auth: `requireMobileAuth` or `requirePhoneBridgeAccess`. Destructive ops add `requireDesktopSuperMasterAuthority`. Imports: phoneMirrorConnector, phoneDeveloperOptionsConnector, phoneTermuxConnector, phoneProMirrorConnector, phoneConstructionBridge, virtualDisplayConnector.

## Route Groups

**Edge Computing (3):** `/edge/status`, `/edge/gemini` POST, `/edge/compute` POST — Gemini-first phone assistant (text, camera vision, live search, IoT planning). Edge compute runs commands on phone via Termux.

**Developer Options (4):** `/developer-options/catalog`, `/developer-options/status`, `/developer-options/set`, `/developer-options/sweep` — ADB-level phone developer option control.

**Pro Mirror (3):** `/pro-mirror/status`, `/pro-mirror/start`, `/pro-mirror/stop` — scrcpy-based high-fidelity phone mirroring.

**Phone Mirror (5):** `/mirror/status`, `/mirror/capture`, `/mirror/image`, `/mirror/voice/latest` — ADB screenshot capture pipeline + voice output retrieval.

**Mirror Control (4):** `/mirror/control/tap`, `/mirror/control/swipe`, `/mirror/control/text`, `/mirror/control/key` — ADB input injection (tap coordinates, swipe gestures, text entry, key events).

**WhatsApp (5):** `/mirror/whatsapp/open`, `/mirror/whatsapp/targets`, `/mirror/whatsapp/send`, `/mirror/whatsapp/linked/open-chat`, `/mirror/whatsapp/linked/voice-note` — open WhatsApp, list contacts, send text, open specific chat, send voice notes via linked device.

**Virtual Display (7):** `/virtual-display/list`, `/virtual-display/create`, `/virtual-display/destroy`, `/virtual-display/capture`, `/virtual-display/image/:displayIndex`, `/virtual-display/tap`, `/virtual-display/type`, `/virtual-display/launch` — headless virtual display management for multi-app scenarios.

## Notable Patterns
- Two mirror systems: standard ADB mirror vs Pro (scrcpy) mirror
- WhatsApp integration spans both direct ADB and linked-device paths
- Virtual display system enables headless multi-app phone automation
- Edge compute via Termux connector runs arbitrary commands on phone
- Gemini API is policy-gated (can be disabled)
