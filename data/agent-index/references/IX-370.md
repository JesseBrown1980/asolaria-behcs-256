## 2. What Healthcare OCR Does That Asolaria's Doesn't

Asolaria already has `tools/caption_ocr_bridge.py` — but it is a **consumer** of the healthcare OCR, not a replacement. The bridge:
- Imports `TextExtractor` directly from `ai_healthcare_project` via sys.path injection (line 189)
- Captures screenshots, runs OCR, pushes caption lines to Asolaria's `/api/voice/bridge-caption/push`
- Handles deduplication, noise filtering, delta detection, session refresh
- Has NO medical parsing or FHIR mapping

**Novel capabilities in healthcare that Asolaria lacks:**
1. **MedicalParser** — regex extraction of 13 lab test types (glucose, HbA1c, cholesterol, LDL, HDL, triglycerides, creatinine, BUN, sodium, potassium, hemoglobin, WBC, platelets), medications with dosage/frequency/route, 5 vital sign types (BP, HR, temp, RR, O2), conditions, dates
2. **FHIRMapper** — LOINC codes for all lab tests and vitals, SNOMED CT codes for 8 conditions, UCUM unit mapping, full FHIR R4 resource generation with provenance linking
3. **Confidence scoring** — per-word confidence from Tesseract, per-detection confidence from EasyOCR, averaged into result
4. **Dual-engine fallback** — if Tesseract fails, falls back to EasyOCR and vice versa
