# Transport Provenance Hardening

## Durable Truth

The transport write path is no longer provenance-blind in code.

Inbound MQTT federation writes now carry a stable envelope with:

- `source_kind`
- `source_topic`
- `host_scope`
- `observed_at`
- `received_at`
- `verifier`
- `transport_epoch`
- `status`
- `evidence_ref`
- `payload_hash`

The remote-node registry stores that envelope per channel and exposes the latest transport source fields in federation summaries.

## Use Rule

Use this entry when reasoning about future transport-backed node facts, but do not treat it as live runtime truth until the protected listener is restarted and fresh telemetry is observed.
