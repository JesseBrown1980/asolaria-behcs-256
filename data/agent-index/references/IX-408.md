During screenshot-first recovery, several plausible screenshot clusters were ruled out because they showed only supporting surfaces, not the actual task/output state Jesse was judging.

Key correction:

- screenshot authority still stands
- but not every screenshot inside the right date range is the right evidence set
- runtime-open screenshots are not enough
- graph-visible screenshots are not enough
- code-fix or terminal-progress screenshots are not enough
- the correct screenshot set must show the actual surface/output Jesse called working, broken, or fixed

Significance:

- prevents false convergence on convincing but wrong screenshot clusters
- distinguishes infrastructure-state evidence from task-truth evidence
- turns screenshot recovery into a stricter filter:
  - first identify candidate clusters by time
  - then reject any cluster that shows only orchestration, support work, credentials, browsing, or side tasks
  - keep only clusters that show the exact judged surface

Practical rule:

- if a screenshot only proves that Asolaria, Claude Code, graph runtime, or a code patch was open, classify it as supporting context
- if a screenshot shows the exact repaired target surface or failure surface Jesse was reacting to, classify it as primary truth
