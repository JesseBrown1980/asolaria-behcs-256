The current QDD proof ladder is not the old promotion labels alone. The trusted order now is:

1. provenance freeze
2. fixture-aligned local proof
3. bounded dev proof
4. lane-specific staging proof
5. cross-lane staging landing
6. prod review
7. prod landing

Why this is required:

- March 27 preserved real bounded branch-local proof
- March 28 showed continuity trust had weakened enough that branch-local summaries could not stand as sole authority
- lane-complete live proof was still missing

Required implication:

- every lane now needs a provenance packet, fixture packet, execution packet, result packet, and promotion packet
- branch-local tests are no longer enough by themselves
- hardware/topology-sensitive lanes must capture SSID, VPN state, bridge path, and control surface explicitly

Current lane placement:

- Queue: between fixture-aligned local proof and bounded dev proof
- NovaLUM: between provenance freeze and fixture-aligned local proof for live sync, but internal-scan authority is already strong
- EZ Protect: historically dev/staging accepted, but currently still blocked before fresh environment-facing proof
