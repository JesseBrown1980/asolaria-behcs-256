After the NovaLUM was reconfigured to the verified Jesse/open state and the desktop app was reopened, repeated live sync attempts produced:

- device UI reported `synchronizing`
- local server accepted repeated direct Wi-Fi connections from `192.168.1.10:2000`
- each connection then ended immediately on the server side with `socket ended`
- the desktop app log did not record a matching new proxy attempt during these direct device Wi-Fi tries

Evidence:
- `D:\projects\QDD\ebacmap-master\reports\novalum-sync-server-20260329-175500-enc.out.log`
- `C:\Users\acer\Downloads\logs\NL2XSync_2026-03-29.log`

Significance:
- the live lane has progressed beyond simple network reachability
- the device itself is attempting direct Wi-Fi sync and considers the session active enough to display `synchronizing`
- the remaining fault is still in the earliest direct device-to-server protocol/handshake stage after connect
