# routes/voice.js — 29 routes (voice/audio pipeline)

Mounted as `/api/voice/*`. Uses `createVoiceRouter(ctx)`. Auth: mixed — some open (devices, status), some `requireMobileAuth`. Imports voice device inventory, transcription, TTS, live capture, bridge-caption systems.

## Route Groups

**Device Inventory (2):** `/devices`, `/status` — enumerate audio capture modes, input devices, FFmpeg availability, selected input/output devices. Status includes voice plan + live capture state.

**Transcription (3):** `/transcribe` POST, `/listen` POST, `/transcribe-chunk` POST — file-based transcription, microphone listen session, chunked streaming transcription.

**Live Capture (5):** `/live/status`, `/live/start`, `/live/stop`, `/live/transcript`, `/live/injection/enqueue`, `/live/injection/clear` — continuous voice capture with start/stop lifecycle. Injection system allows inserting synthetic utterances into the live transcript stream.

**Bridge Caption Push (3):** `/bridge-caption/push` GET+POST, `/bridge-caption/lane-test` POST — external caption sources (Google Meet captions) push text into Asolaria. Lane-test validates routing. Both GET and POST accepted on push endpoint.

**Bridge Caption Pull (4):** `/bridge-caption/reset`, `/bridge-caption/pull`, `/bridge-caption/lanes`, `/bridge-caption/auth-status` — consumers pull accumulated captions, view lane summaries, check push auth status.

**Bridge Caption Orchestrator (7):** `/bridge-caption/orchestrator/status`, `/bridge-caption/orchestrator/start`, `/bridge-caption/orchestrator/stop`, `/bridge-caption/orchestrator/reset`, `/bridge-caption/orchestrator/assign`, `/bridge-caption/orchestrator/route`, `/bridge-caption/orchestrator/analysis`, `/bridge-caption/orchestrator/tick` — multi-lane caption orchestration: assign speakers to lanes, route output targets, run analysis passes, manual tick.

**TTS & Reply (3):** `/speak` POST, `/latest` GET, `/reply` POST — text-to-speech generation, get latest voice output, voice reply (TTS + response generation).

## Notable Patterns
- Bridge-caption system is a full meeting transcription pipeline (Google Meet → lanes → orchestrator → analysis)
- Orchestrator supports multi-speaker lane assignment and routing to different output targets
- Live capture injection enables synthetic utterance testing
- Push endpoint accepts both GET and POST for flexible integration
- Voice plan system configures the full audio pipeline (capture mode, device, TTS engine)
