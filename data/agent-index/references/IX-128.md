# Flash Attention on Smaller GPU Computer Systems
**Reference image from Plan B archive (PNG)**

## Key Findings
- Flash Attention rewrites the attention computation to be IO-aware — it tiles the Q/K/V matrices to fit in GPU SRAM, avoiding repeated reads from slow HBM
- Achieves 2-4x speedup over standard attention with no approximation — exact same output, just faster memory access patterns
- Critical for small GPUs (24GB VRAM class): reduces peak memory from O(N^2) to O(N) for sequence length N
- Enables processing of longer sequences on consumer hardware that would otherwise OOM with standard attention
- Combined with Mamba (IX-122), represents the two main paths to efficient long-context processing: avoid attention entirely (Mamba) or make attention efficient (Flash)

## How This Connects to Asolaria
- Directly enables Asolaria to run on Jesse's home hardware — Flash Attention is what makes large context windows feasible on a 24GB GPU
- Any local model serving (via S-LoRA, IX-123) will need Flash Attention as the attention backend
- The "home computer civilization" concept requires these efficiency techniques to be viable

## Potential Application
When deploying local models for Asolaria agents, Flash Attention should be mandatory in the serving stack — it is the difference between running one agent and running a full civilization on the same hardware.
