User-provided physical photo confirms the intermediary hardware in the NovaLUM USB path is labeled `NL2 INTERFACE ADAPTER`.

This materially strengthens the live-device model:

- Windows sees a Microchip-backed `USB Serial Port (COM9)`
- the physical bridge is not generic or imagined
- the serial path is routed through dedicated NL2 adapter hardware before it reaches the NovaLUM

Practical consequence:

- direct native-USB assumptions should be dropped
- the correct mental model is:
  - host computer
  - NL2 interface adapter
  - NovaLUM connector path
- local control should prefer protocol-aware serial or desktop-app-mediated operation over arbitrary USB probing
