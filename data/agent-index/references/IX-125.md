# GPT Is Becoming a Turing Machine: Here Are Some Ways to Program It
**Schuurmans, Google Research, 2023**

## Key Findings
- Demonstrates that sufficiently large LLMs can simulate Turing machine computations through carefully structured prompts
- Shows LLMs executing algorithms: sorting, graph traversal, dynamic programming — not by memorization but by following procedural steps encoded in the prompt
- Introduces "memetic proxy programming": encoding algorithms as in-context demonstrations that the model follows as a virtual instruction set
- The key insight is that LLMs can be PROGRAMMED, not just PROMPTED — the distinction matters for building reliable agent systems
- Performance scales with model size and prompt engineering quality, not fine-tuning

## How This Connects to Asolaria
- Validates the Asolaria approach of using structured prompts (bootstrap-prompt.txt, IX chains) as a programming language for agent behavior
- Helm's bootstrap prompt IS a program in this framework — it encodes the simulation rules the agent must follow
- The IX system functions as a shared instruction set that all agents can reference and execute

## Potential Application
Asolaria's agent prompts should be treated as programs, not suggestions — designing them with algorithmic precision (clear state, transitions, halting conditions) will produce more reliable agent behavior than conversational prompting.
