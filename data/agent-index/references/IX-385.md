## How It Connects to Asolaria
MQTT WebSocket bridge — SwarmDesk loads mqtt.js from CDN and connects to Asolaria's broker:
- **WSS:** `wss://127.0.0.1:18885/mqtt` (preferred, TLS)
- **WS:** `ws://127.0.0.1:18884/mqtt` (fallback, plain)
- **Mode:** `hybrid_mqtt_http` — MQTT for realtime, HTTP for initial state

## MQTT Topics Subscribed
| Topic | What It Shows |
|-------|---------------|
| `asolaria/test/local/world/state` | Civilization world state (districts, entities) |
| `asolaria/test/local/agents/colony/status` | Colony agent status |
| `asolaria/test/local/runtime/health` | Server health snapshot |
| `asolaria/test/local/graph/events` | Graph events stream |

## Architecture
1. Dashboard loads in browser (madnessinteractive.cc or local)
2. `window.SwarmDeskDashboard` — global state: currentProject, activeAgent, realtime status
3. MQTT client subscribes to 4 Asolaria topics, receives retained + live updates
4. Three.js renders 3D world with project workstations, agent entities, civilization buildings
5. `applyRealtimeState()` pushes MQTT data into the 3D scene
6. Sidebar callbacks: onProjectSelect, onAgentInteract, onMCPTool, onActivityLog

## Key Insight
SwarmDesk is a VIEWER (IX-046). It reads Asolaria's state via MQTT but doesn't write back. The 3D world is a projection of the civilization model, not the model itself.
