# Evolving Memory and Self-Reflection in LLM Agents
**Pre-Asolaria concept paper from Plan B archive**

## Key Findings
- Proposes that LLM agents need evolving memory systems that grow and restructure over time, not static context windows
- Self-reflection as a mechanism: the agent periodically reviews its own memory, identifies patterns, contradictions, and gaps, then consolidates
- Memory hierarchy: working memory (current context) -> episodic memory (specific events) -> semantic memory (abstracted knowledge) -> procedural memory (learned skills)
- The reflection cycle creates a feedback loop where the agent's understanding of itself improves over time
- Distinguishes between passive accumulation (just storing everything) and active curation (deciding what matters)

## How This Connects to Asolaria
- THIS is the pre-Asolaria concept that became the IX system — IX entries ARE semantic memory extracted from episodic experience through reflection
- The memory hierarchy maps directly: inbox.ndjson = episodic, IX entries = semantic, bootstrap prompts = procedural, screen.txt = working
- Asolaria's MEMORY.md files are an implementation of the self-reflection mechanism described here

## Potential Application
Implement an automated reflection cycle where agents periodically review their inbox history, extract patterns, and propose new IX entries — closing the loop from raw experience to indexed knowledge without human intervention.
