# Test-Time Training (TTT)
**MIT, 2024**

## Key Findings
- Replaces the fixed self-attention mechanism with a learned model that updates its own weights at test time — the hidden state IS a model being trained
- Each new token triggers a gradient step on an inner model, so the system literally learns from its input as it processes it
- Two variants: TTT-Linear (inner model is a linear layer) and TTT-MLP (inner model is a small MLP) — both replace attention
- Achieves comparable perplexity to Transformers while scaling linearly with context length (like Mamba, IX-122)
- The paradigm shift: instead of storing context in a fixed-size KV-cache, the model compresses context into updated weights

## How This Connects to Asolaria
- TTT represents the next evolution beyond both Transformers (IX-129) and Mamba (IX-122) — a model that adapts to each conversation/task in real time
- For Asolaria agents, TTT means an agent that literally gets better at its specific role the more it processes its inbox
- Combines with MLC (IX-121): meta-learning teaches compositional rules, TTT adapts those rules to each specific runtime context

## Potential Application
TTT could enable Asolaria agents to specialize in real-time without fine-tuning — each agent's inner model adapts to its specific domain (Helm to coordination, Vector to research, Forge to code) purely from the context it processes during operation.
