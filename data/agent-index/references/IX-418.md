Two different recovery lanes must not be conflated:

- Queue local re-proof
  - localhost Mongo
  - localhost FTP
  - no real device required
  - VPN and Wi-Fi are mostly irrelevant

- NovaLUM live sync
  - real device present
  - Wi-Fi and/or USB bridge path matters
  - encryption service matters
  - same-LAN visibility can matter
  - VPN can interfere with routing, DNS, adapter priority, or local reachability

Operational rule:

- keep Queue re-proof strictly localhost-only
- treat NovaLUM live sync as a topology-sensitive lane
- for NovaLUM live testing, record:
  - active SSID
  - whether the computer and device are on the same intended network
  - whether VPN is enabled
  - whether the path is USB-assisted, Wi-Fi-assisted, or both

User memory worth preserving:

- earlier working lane likely involved the open `Jesse` Wi-Fi
- `Jesse 5 g` with password may not be equivalent
- the device may depend on the originally working network context rather than any arbitrary current connection
