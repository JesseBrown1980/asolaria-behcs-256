The repo preserves a real local Queue re-proof candidate:

- multiple seed scripts target `mongodb://localhost:27017/ebacmap-local-test`
- `seed-dev-user.js` and `seed-facilities.js` create the client, location, permissions, and FTP config shape needed for local replay
- the Tier 1 Queue CSV already exists

But the current local wiring is not fully self-consistent:

- `apps/web-ebacmap/src/pages/api/test-results/ftp-import.ts` derives `env=local` when `APP_URL` points at localhost
- `libs/utils/src/lib/queue.ts` maps empty or localhost URLs to `local`
- `apps/queue-server/src/index.ts` in `APP_ENV=local` mode only boots a `staging` queue worker backed by `STG_MONGODB_URI`

So the practical local Queue re-proof path is almost recoverable, but still needs one env-slot fix:

- either boot a real `local` queue worker against `ebacmap-local-test`
- or deliberately map the local web app into the `staging` slot used by the local queue worker

This is the key technical reason Queue is locally recoverable but not yet one-command re-playable.
