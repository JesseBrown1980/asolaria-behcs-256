# IX-471 — Under-the-Metal Sovereignty Observers

9 shell scripts on Falcon USB (`E:\sovereignty\boot\`) collect bare-metal Linux telemetry as structured JSON into `E:\sovereignty\observations\`. All operations are READ-ONLY. Nothing is written to hardware.

## The 9 Observers

### 1. hardware-observer.sh
**Collects:** Full hardware inventory via dmidecode (BIOS/board/chassis), lspci (PCI topology), /proc/cpuinfo (CPU model/flags/cores), /proc/meminfo (RAM), lsblk (block devices with model/serial/fs), lsusb (USB tree), thermal zones, and MSR 0x13A (IA32_FEATURE_CONTROL / Boot Guard).

### 2. network-observer.sh
**Collects:** ip addr (all interfaces with IPs), ip route (routing table), MAC addresses from /sys/class/net, wireless state (iwconfig/iw), DNS configuration (resolv.conf + systemd-resolved).

### 3. thermal-observer.sh
**Collects:** Thermal zones (/sys/class/thermal — type and temp per zone), hwmon temperatures (per-sensor with labels), fan speeds (RPM per fan with labels), CPU frequency governors (current/min/max MHz and governor policy per core).

### 4. gpu-observer.sh
**Collects:** DRM card inventory (/sys/class/drm — vendor, device, driver, Intel GT frequencies, power state), hwmon GPU sensors (temperatures in Celsius, power draw in watts).

### 5. msr-observer.sh
**Collects:** Model-Specific Registers via rdmsr (requires root + msr-tools):
- **0x13A** — Boot Guard / SACM_INFO status
- **0x3A** — IA32_FEATURE_CONTROL: lock bit, VMX in/out SMX, SGX enable/launch
- **0x1A0** — IA32_MISC_ENABLE: SpeedStep, XD bit disable, turbo disable
- **0x610** — MSR_PKG_POWER_LIMIT: PL1/PL2 watts, lock bit (BIOS-locked power limits)

### 6. iommu-observer.sh
**Collects:** IOMMU groups with per-device BDF/description/driver, DMAR table presence, DMA remapping dmesg, Intel VT-d cmdline flag, AMD-Vi detection, IOMMU driver identity, full kernel cmdline.

### 7. firmware-observer.sh
**Collects:** UEFI boot entries (efibootmgr -v), Secure Boot state (efivar byte read + mokutil), BIOS vendor/version/date and system manufacturer/product (dmidecode), Intel ME status (/dev/mei0 + mei-amt-check/intelmetool), kernel cmdline.

### 8. storage-observer.sh
**Collects:** Block device detail (lsblk with UUID/ROTA/RO/DISC-MAX), partition tables (fdisk -l), filesystem IDs (blkid), SMART health per disk (smartctl -H with model/serial), NVMe device list (nvme-cli), mount state.

### 9. sovereignty-init.sh
**Collects:** Master boot orchestrator. Runs all 7 domain observers in sequence, verifies USB mount, installs Node via nvm if missing, then boots the sovereignty kernel (sovereignty-boot.js) with ASOLARIA_SOVEREIGNTY_ROOT pointed at the USB. Produces a full status report with pass/fail per observer and kernel boot state.

## Output format
Each observer writes timestamped JSON to `E:\sovereignty\observations\` (e.g., `hardware-20260330-143000.json`). sovereignty-init.sh also writes a unified log.
