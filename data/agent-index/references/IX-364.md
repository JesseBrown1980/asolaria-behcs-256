## Architecture (7 major subsystems)

### 1. FHIR Connector (backend/fhir_connector.py)
- OAuth2/API key auth, paginated bundle fetching, FHIR resource validation
- Supports Epic and Cerner SMART-on-FHIR sandboxes
- HTTP client with SOCKS proxy support (httpx)

### 2. LLM Engine + RAG Fusion (backend/llm_engine.py, backend/rag_fusion.py)
- Multi-provider (GPT-4, LLaMA-2, Anthropic)
- RAG-Fusion for medical literature retrieval with citations
- Algorithm of Thought reasoning (backend/aot_reasoner.py)

### 3. S-LoRA Manager (backend/s_lora_manager.py)
- Multi-specialty LoRA adapters (cardiology, oncology, etc.)
- Dynamic adapter composition and switching
- Memory-efficient fine-tuning without retraining base models

### 4. MLC Learning + RL Agent (backend/mlc_learning.py, backend/rl_agent.py)
- Meta-Learning for Compositionality — online learning from clinician feedback
- Reinforcement learning for adaptive risk classification (experiments/rl_policy_demo.py)
- Feedback-to-policy loop with clinical caution constraints

### 5. GNN Anomaly Detection (backend/anomaly_detector/)
- 4 architectures: baseline, contrastive (94.71%), prototype (94.24%), GSL-GNN (96.66% best)
- Edge-level graph neural networks for API/FHIR log security anomaly detection
- Uses torch_geometric (GCNConv), attention-based graph structure learning
- Clinical graph builder for patient-level anomaly detection
- Files: models/gnn_baseline.py, contrastive_gnn.py, prototype_gnn.py, gsl_gnn.py, graph_builder.py, clinical_graph_builder.py

### 6. OCR Pipeline (backend/ocr/)
- Dual engine: tesseract_processor.py (printed text) + easyocr_processor.py (handwritten/complex)
- Medical parser: extracts lab values, medications, vital signs, conditions, dates from OCR text
- FHIR mapper: converts extracted medical data to FHIR resources
- Document endpoints integrated into API

### 7. Compliance & Security
- HIPAA: crypto-shredding for PII in audit logs (patient-specific encryption keys)
- GDPR: right to be forgotten, consent management API, data deletion service
- SOC2: audit logging (audit-logs/ with daily JSONL files, Jan-Feb 2026)
- Security: CSP headers, XSS/SQLi detection, input validation
- Compliance keys: compliance_keys.json — per-entity encryption keys (test users, patients, SQL injection test strings)
- Docs: docs/compliance/ (SOC2 checklist, risk assessment, security architecture), docs/regulatory/ (risk management, software verification)

## API Endpoints (FastAPI on port 8000)
- POST /api/v1/analyze-patient — full patient analysis with risk scores
- GET /api/v1/patient/{id}/fhir — FHIR bundle fetch
- POST /api/v1/feedback — clinician feedback for MLC learning
- POST /api/v1/query — medical question answering with RAG
- GET /api/v1/adapters — S-LoRA adapter status
- GET /api/v1/patients/dashboard — multi-patient dashboard (polling)
- WS /ws/patient-updates — real-time WebSocket updates
- POST /api/v1/register-device — FCM push notification registration
- POST /api/v1/cache/clear — flush analysis cache
