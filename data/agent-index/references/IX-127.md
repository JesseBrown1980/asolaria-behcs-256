# MAC-SQL: Multi-Agent Collaboration for Text-to-SQL
**Wang et al., 2023**

## Key Findings
- Decomposes text-to-SQL into specialized agent roles: Selector (picks relevant tables/columns), Decomposer (breaks complex queries into sub-queries), and Refiner (fixes SQL errors via execution feedback)
- Multi-agent approach outperforms single-agent prompting on complex queries by 2-5% on BIRD benchmark
- The Selector agent dramatically reduces the schema context each downstream agent must process — a form of attention focusing
- Error correction through the Refiner agent creates a self-healing pipeline — bugs are caught by execution and fixed in-loop
- Shows that agent specialization + collaboration beats a single generalist agent on structured reasoning tasks

## How This Connects to Asolaria
- MAC-SQL validates Asolaria's multi-agent architecture: specialized agents (Helm=coordinator, Forge=builder, Vector=brain, Rook=ops) outperform a single do-everything agent
- The Selector/Decomposer/Refiner pipeline mirrors how Helm decomposes tasks and delegates to specialized agents
- The self-healing Refiner pattern should be adopted by Asolaria agents — execute, check, fix, re-execute

## Potential Application
Apply the MAC-SQL decomposition pattern to Asolaria's task routing: Helm selects the relevant agent, the agent decomposes the task, and a verification step confirms the output before delivery — reducing errors in the agent pipeline.
