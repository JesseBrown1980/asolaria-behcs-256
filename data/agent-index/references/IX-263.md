# Liris Startup Readiness Split

## Durable Truth

`Liris part of start up response.docx` says startup claims must be split into:

- control-plane readiness
- main `4781` proof

and must remain tied to the target host's own live probes.

On the Rayssa host, the recorded result was `ready_for_owner_restart`, not startup completion.

## Use Rule

Use this entry when interpreting startup claims from relay packets.

No startup statement should be promoted across hosts unless the same listener and status are observable on the host being discussed.
