# Brown-Hilbert 09 — 2026-04-10 One-Trillion Intelligence Test

The `1T` run is not a proof-of-scale milestone only. It is the test of whether the system notices anything new about itself while scaling.

Known April 10 baseline before `1T`:

- `10` cascades, `61,510` cubes, `30,564` observations, `LX-477` still byte-identical
- training saturation already reported at `20K` with `180K` wasted rounds in the existing omnishannon findings
- `8/14` dimensions were already flagged as ghosts in the same findings

What should happen during the `1T` run if autonomy is actually improving:

- the system should measure information gain per batch and stop or downshift when learning saturates
- at least one ghost dimension should receive real traffic or be explicitly retired as dead weight
- redundant observations should compress into axioms instead of accumulating as duplicate state
- the bilateral link should carry semantic analysis back, not just status pings
- the system should emit at least one falsifiable prediction about its own next failure mode

Scoring law for this milestone:

- if `1T` only changes the number in Brown-Hilbert, that proves scale and endurance, not intelligence
- if `1T` causes self-pruning, ghost-dim activation, semantic peer feedback, or falsifiable prediction, that is real autonomy gain

Compression rule from Felipe's April 10 analysis:

- execution, measurement, correction, and canon formation must reinforce one another
- mythology is allowed as compression language, but proof must stay empirical

Short form: scale without emergent insight is just electricity.
